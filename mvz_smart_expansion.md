# MVZ 智能展开功能说明

## 功能概述

为 MVZ 节点的文件树实现了智能展开机制，让用户在视野内能看到最重要的压缩包和文件。

## 智能展开算法

### 核心策略

1. **优先展开文件少的压缩包**
   - 按文件数量排序，文件少的压缩包优先展开
   - 更容易浏览和理解

2. **自动展开单子节点文件夹**
   - 只有一个子节点的中间文件夹自动展开
   - 减少不必要的层级，直达内容

3. **智能折叠多子节点文件夹**
   - 有多个子节点的文件夹保持折叠
   - 避免信息过载

4. **可见文件数量限制**
   - 默认最多展示 50 个文件
   - 超过限制的压缩包只展开到压缩包层级

5. **确保压缩包可见**
   - 自动展开从压缩包到根的路径
   - 保证用户能看到所有压缩包

## 使用方式

### 自动智能展开
- 添加文件后自动应用智能展开
- 首次加载时自动展开

### 手动控制按钮

1. **✨ 智能展开**（黄色星星图标）
   - 应用智能展开算法
   - 重新计算最佳展开状态

2. **⌄ 全部展开**（向下箭头）
   - 展开所有节点
   - 查看完整树结构

3. **› 全部折叠**（向右箭头）
   - 折叠所有节点
   - 回到初始状态

4. **📋 复制路径**
   - 复制所有文件路径到剪贴板

## 技术实现

### 模块化设计

- `treeUtils.ts` (254 行)：树构建和智能展开算法
- `MvzNode.svelte` (632 行)：UI 组件
- 符合单文件不超过 800 行的规范

### 核心函数

1. **buildCompleteFileTree()**
   - 构建完整的文件树结构
   - 支持 Windows 和 Unix 路径
   - 递归构建外部和内部树

2. **calculateSmartExpansion()**
   - 智能展开算法核心
   - 返回应该展开的节点路径集合

3. **countFiles()**
   - 计算节点下的文件总数
   - 用于排序和限制

4. **expandPathToRoot()**
   - 展开从节点到根的路径
   - 确保节点可见

5. **expandArchiveContent()**
   - 智能展开压缩包内部内容
   - 应用单子节点展开策略

## 示例场景

### 场景 1：少量文件
```
输入：3 个压缩包，每个 5 个文件
结果：全部展开，显示所有文件
```

### 场景 2：大量文件
```
输入：10 个压缩包，每个 20 个文件
结果：只展开前 2-3 个压缩包（约 50 个文件）
其他压缩包折叠，但路径可见
```

### 场景 3：深层嵌套
```
输入：压缩包内有多层单子节点文件夹
结果：自动展开到实际内容层级
D:/archive.zip/folder1/folder2/folder3/file.txt
→ 自动展开到 file.txt 可见
```

### 场景 4：混合结构
```
输入：有的文件夹有多个子节点，有的只有一个
结果：单子节点自动展开，多子节点保持折叠
```

## 用户体验优化

1. **首次加载自动展开**
   - 添加文件后自动应用智能展开
   - 无需手动操作

2. **状态持久化**
   - 手动展开/折叠的状态会保留
   - 直到重新应用智能展开

3. **视觉反馈**
   - 日志显示展开的节点数量
   - 清晰的图标指示

4. **灵活控制**
   - 可以随时切换展开模式
   - 支持手动微调

## 性能考虑

- 算法复杂度：O(n log n)，n 为文件数
- 适用于数千个文件的场景
- 展开状态使用 Set 存储，查询 O(1)

# Findz 图片元数据功能集成说明

## 概述

已成功将 `PackU/AutoUnzip/src/findz` 中的图片元数据功能集成到 aestivus 的 findz 节点中，并添加了多路径输入支持。

## 新增功能

### 1. 图片元数据支持

支持读取和过滤图片的元数据信息（宽度、高度、分辨率等）。

### 2. 多路径输入

支持同时搜索多个目录，结果自动合并输出。

## 修改内容

### 1. 后端适配器 (`src-python/adapters/findz_adapter.py`)

#### 新增功能
- **输入参数**：添加 `with_image_meta: bool` 参数，用于启用图片元数据读取
- **模块导入**：导入 `IMAGE_FIELDS` 常量
- **元数据启用**：在搜索时调用 `FileInfo.enable_image_meta(True)` 启用功能
- **数据转换**：`_file_info_to_dict()` 方法支持返回图片元数据字段

#### 支持的图片元数据字段
- `width`: 图片宽度（像素）
- `height`: 图片高度（像素）
- `resolution`: 分辨率字符串（如 "1920x1080"）
- `megapixels`: 百万像素数
- `aspect_ratio`: 宽高比

### 2. 前端类型定义 (`src/lib/components/nodes/findz/types.ts`)

#### FileData 接口扩展
```typescript
export interface FileData {
  // ... 原有字段

  width?: number;
  height?: number;
  resolution?: string;

```

#### FindzNodeState 接口扩展
```typescript
export interface FindzNodeState {
  // ... 原有字段
  targetPath: string; // 保留用于单路径兼容
  targetPaths: string[]; // 多路径列表
  withImageMeta: boolean; // 是否启用图片元数据
}
```

### 3. Findz 节点组件 (`src/lib/components/nodes/findz/FindzNode.svelte`)

#### 新增功能
- **多路径输入**：支持添加多个搜索路径
  - 输入框支持多行粘贴（textarea 自动调整高度）
  - 文件夹选择对话框
  - **智能剪贴板粘贴**：
    - 自动识别单个或多个路径
    - 支持 Windows 资源管理器复制格式
    - 自动去除引号和空白字符
    - 自动去重
  - 路径列表显示（可删除单个或清空全部）
- **图片元数据开关**：在过滤器区块添加复选框控制
- **API 调用**：执行搜索时传递 `paths` 数组和 `with_image_meta` 参数
- **结果显示**：文件列表中显示图片分辨率信息
- **自动迁移**：旧的单路径配置自动迁移到多路径列表

#### UI 改进
- 路径输入框改为 textarea，支持多行显示
- 输入框高度自动调整（最大 100px）
- 路径列表支持滚动显示
- 每个路径可单独删除
- 显示路径数量统计
- 空状态提示

### 4. 过滤器构建器 (`src/lib/components/nodes/findz/FilterBuilder.svelte`)

#### FilterConfig 接口扩展
```typescript
interface FilterConfig {
  // ... 原有字段

  widthMax: string;

```

#### 新增 UI 组件
- **图片尺寸区块**：可视化配置图片宽度和高度范围
- **分辨率预设**：快速选择常见分辨率（1080p, 4K, 8K）
- **自定义范围**：支持自定义宽度和高度的最小/最大值

#### 内置预设
新增预设：**压缩包内高度630**
- 搜索压缩包内高度为 630 像素的图片
- 自动启用图片元数据
- 设置高度范围为 630-630

#### SQL 生成
支持生成图片元数据相关的 SQL 条件：
```sql
-- 宽度范围
width BETWEEN 1920 AND 3840
width >= 1920
width <= 3840

-- 高度范围
height BETWEEN 1080 AND 2160
height >= 630
height <= 1080

-- 分辨率预设
resolution = "1920x1080"
resolution = "3840x2160"
resolution = "7680x4320"
```

## 使用方法

### 1. 添加搜索路径

有三种方式添加路径：

1. **手动输入**：在输入框中输入路径，按 Enter 或点击 + 按钮添加
   - 支持单行输入单个路径
   - 支持多行粘贴多个路径（自动识别）
   - 按 Shift+Enter 可以换行输入多个路径
   
2. **文件夹选择**：点击文件夹图标，通过对话框选择目录

3. **剪贴板粘贴**：点击剪贴板图标，自动识别并添加路径
   - **单个路径**：直接放入输入框
   - **多个路径**：自动批量添加到列表
   - 支持 Windows 资源管理器复制的格式（带引号、换行分隔）
   - 自动去除引号和空白字符
   - 自动去重

#### 智能路径识别

系统会自动识别以下格式的路径：
```
"C:\Users\Documents\Pictures"
"D:\Photos\2024"
"E:\Archives"
```

或者：
```
C:\Users\Documents\Pictures
D:\Photos\2024
E:\Archives
```

**使用场景示例**：
1. 在 Windows 资源管理器中选择多个文件夹
2. 按 Ctrl+C 复制
3. 在 Findz 节点中点击剪贴板图标
4. 所有路径自动添加到列表中

可以添加多个路径，搜索结果会自动合并。

### 2. 启用图片元数据

在 Findz 节点的过滤器区块中，勾选"启用图片元数据"复选框。

### 3. 使用可视化模式

1. 勾选"图片尺寸"选项
2. 选择分辨率预设（1080p/4K/8K）或自定义范围
3. 输入宽度/高度的最小值和最大值

### 4. 使用 SQL 模式

切换到 SQL 模式，直接输入条件：
```sql
-- 查找高度为 630 的图片
height = 630

-- 查找宽度大于 1920 的图片
width > 1920

-- 查找 1080p 分辨率的图片
resolution = "1920x1080"

-- 组合条件：压缩包内高度为 630 的图片
archive <> "" AND height = 630

-- 查找宽高比接近 16:9 的图片
aspect_ratio BETWEEN 1.7 AND 1.8
```

### 5. 使用预设

点击"预设"按钮，选择"压缩包内高度630"预设，自动配置所有参数（包括启用图片元数据）。

### 6. 多路径搜索示例

**方式一：逐个添加**
1. 输入 `D:\Pictures`，按 Enter
2. 输入 `E:\Photos`，按 Enter
3. 输入 `F:\Archives`，按 Enter

**方式二：批量粘贴（推荐）**
1. 在 Windows 资源管理器中选择多个文件夹
2. 按 Ctrl+C 复制
3. 在 Findz 节点中点击剪贴板图标
4. 所有路径自动添加

**方式三：多行输入**
1. 在输入框中粘贴或输入：
   ```
   D:\Pictures
   E:\Photos
   F:\Archives
   ```
2. 按 Enter 或点击 + 按钮
3. 三个路径自动识别并添加

然后设置过滤条件（如：`ext = "jpg" AND size > 1M`），点击搜索，三个目录的结果会合并显示。

## 技术细节

### 智能路径解析

`parseMultiplePaths()` 函数支持：
- 按行分割路径
- 自动去除双引号和单引号
- 验证路径格式：
  - Windows 绝对路径：`C:\` 或 `C:/`
  - Unix 绝对路径：`/`
  - 相对路径：`./` 或 `../`
- 自动去重和空白处理

### 图片元数据读取

- **文件系统文件**：使用 `get_image_dimensions_cached()` 带缓存读取
- **压缩包内文件**：使用 `get_image_dimensions_from_bytes()` 从字节流读取
- **支持格式**：
  - imagesize 支持：jpg, png, gif, webp, tiff, svg 等
  - Pillow 支持：jxl, avif, heic, bmp 等

### 性能优化

- **延迟加载**：只在需要时读取图片元数据
- **缓存机制**：文件系统文件使用 LRU 缓存
- **混合策略**：优先使用快速的 imagesize，回退到 Pillow

### 依赖要求

Python 包（在 findz 中）：
```bash
pip install imagesize pillow pillow-avif-plugin pillow-jxl-plugin
```

## 注意事项

1. **启用开关**：必须在节点中启用"图片元数据"才能使用相关字段
2. **性能影响**：启用图片元数据会增加搜索时间（需要读取图片文件）
3. **格式支持**：某些格式需要额外的 Pillow 插件
4. **压缩包编码**：ZIP 文件的中文文件名使用 cp437->gbk 转换处理

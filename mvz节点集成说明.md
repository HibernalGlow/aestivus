# Mvz 节点集成说明

## 概述

已成功将 `PackU/AutoUnzip/src/mvz` 集成到 aestivus，提供压缩包文件操作功能。

## 功能特性

### 支持的操作

1. **提取 (Extract)**
   - 从压缩包中提取指定文件
   - 支持保持目录结构或扁平化提取
   - 可选择输出目录或就近提取

2. **删除 (Delete)**
   - 从压缩包中删除指定文件
   - 批量删除多个文件
   - 支持预览模式

3. **移动 (Move)**
   - 从压缩包中移动文件到目录
   - 实际操作：提取 + 删除
   - 确保提取成功后才删除

4. **重命名 (Rename)**
   - 使用正则表达式批量重命名
   - 支持复杂的模式匹配和替换
   - 在压缩包内部重命名，不解压

### 核心特性

- **批量处理**：支持同时处理多个压缩包
- **预览模式**：dry_run 模式查看操作结果
- **实时进度**：WebSocket 实时显示操作进度
- **智能输入**：支持从 findz 输出直接粘贴
- **错误处理**：详细的错误信息和日志

## 使用方法

### 1. 从 Findz 获取输入

1. 在 Findz 节点中搜索压缩包内的文件
2. 复制搜索结果（格式：`archive_path//internal_path`）
3. 在 Mvz 节点的输入框中粘贴

### 2. 选择操作类型

- **提取**：将文件从压缩包中提取到目录
- **删除**：从压缩包中删除文件
- **移动**：提取文件并从压缩包中删除
- **重命名**：在压缩包内重命名文件

### 3. 配置选项

#### 提取/移动选项
- **输出目录**：指定提取的目标目录
- **就近提取**：提取到压缩包所在目录
- **自动目录**：创建以压缩包命名的子目录
- **扁平化**：不保留目录结构，所有文件提取到同一目录

#### 重命名选项
- **正则模式**：匹配文件名的正则表达式
- **替换为**：替换后的文件名模式

#### 通用选项
- **预览模式**：勾选后只显示操作结果，不实际执行

### 4. 执行操作

点击"执行"按钮，查看实时进度和结果。

## 使用示例

### 示例 1：提取压缩包内的图片

1. Findz 搜索：`ext in ("jpg", "png") AND archive <> ""`
2. 复制结果到 Mvz
3. 选择"提取"操作
4. 设置输出目录：`D:\Extracted`
5. 勾选"自动目录"
6. 执行

### 示例 2：删除压缩包内的临时文件

1. Findz 搜索：`name like "%.tmp" AND archive <> ""`
2. 复制结果到 Mvz
3. 选择"删除"操作
4. 勾选"预览模式"查看将删除的文件
5. 取消预览模式，执行删除

### 示例 3：批量重命名

1. Findz 搜索：`ext = "jpg" AND archive <> ""`
2. 复制结果到 Mvz
3. 选择"重命名"操作
4. 正则模式：`IMG_(\d+)`
5. 替换为：`Photo_$1`
6. 执行（将 IMG_001.jpg 重命名为 Photo_001.jpg）

### 示例 4：移动文件出压缩包

1. Findz 搜索：`size > 10M AND archive <> ""`
2. 复制结果到 Mvz
3. 选择"移动"操作
4. 设置输出目录：`D:\Large_Files`
5. 勾选"就近提取"
6. 执行（文件被提取并从压缩包中删除）

## 技术细节

### 依赖要求

- **7-Zip**：必须安装 7-Zip 并添加到 PATH
- **Python 包**：mvz 模块（已包含在 PackU/AutoUnzip 中）

### 输入格式

文件路径格式：`archive_path//internal_path`

示例：
```
D:\Archives\photos.zip//2024/IMG_001.jpg
D:\Archives\photos.zip//2024/IMG_002.jpg
E:\Backup\docs.7z//reports/2024.pdf
```

### 操作流程

1. **解析输入**：将输入文件列表解析为 ArchiveEntry 对象
2. **按压缩包分组**：将文件按所属压缩包分组
3. **批量执行**：对每个压缩包执行操作
4. **结果汇总**：统计成功和失败的文件数

### 错误处理

- 7-Zip 未找到：提示安装 7-Zip
- 压缩包不存在：跳过并记录错误
- 操作失败：记录详细错误信息
- 部分失败：继续处理其他文件

## 与 Findz 的配合

### 工作流示例

```
Findz 节点 → Mvz 节点
```

1. **Findz**：搜索压缩包内的文件
   - 设置过滤条件
   - 启用图片元数据（如需要）
   - 执行搜索

2. **Mvz**：操作搜索结果
   - 从 Findz 复制结果
   - 选择操作类型
   - 配置选项
   - 执行操作

### 典型场景

1. **清理压缩包**
   - Findz 查找临时文件、缓存文件
   - Mvz 批量删除

2. **提取特定文件**
   - Findz 查找特定类型或大小的文件
   - Mvz 提取到指定目录

3. **整理压缩包**
   - Findz 查找需要重命名的文件
   - Mvz 批量重命名

4. **迁移文件**
   - Findz 查找需要移出的文件
   - Mvz 移动到文件系统

## 注意事项

1. **备份重要数据**：操作会修改压缩包，建议先备份
2. **预览模式**：首次使用建议先用预览模式查看结果
3. **7-Zip 依赖**：确保 7-Zip 已正确安装
4. **路径格式**：输入必须是 `archive//internal` 格式
5. **批量操作**：大量文件操作可能需要较长时间
6. **编码问题**：中文文件名可能需要特殊处理
